Mouse configuration

CPI / DPI

DPI stands for Dots per Inch, while CPI stands for Counts per Inch. CPI or DPI is a way for the mouse to track length. Note that they are different attributes of a sensor but in common context is virtually interchangeable. It is a part of what some refer to as ‘Real sensitivity’. DPI * in-game sensitivity * Windows sensitivity = real sensitivity.

Windows sensitivity

Windows sensitivity is a multiplier to the DPI of the mouse. Below is a table of what each notch does. The default setting is 6/11, and is advised to use for optimal performance.

6/11 is not ‘better’ it is more consistent and easier for the computer to handle without skipping anything, that said, some are still better than others to use.

Below is a table of the different notches:

1/11 = 0.03125 (sub optimal) 2/11 = 0.0625 (sub optimal) 3/11 = 0.25 (sub optimal) 4/11 = 0.5 (sub optimal) 5/11 = 0.75 (every 4th count is ignored = inconsistent) 6/11 = 1.0 (optimal setting) 7/11 = 1.5 (every 2nd count registers as 2 counts of movement = inconsistent + unit skipping) 8/11 = 2.0 (unit skipping) 9/11 = 2.5 (inconsistent + unit skipping) 10/11 = 3.0 (unit skipping) 11/11 = 3.5 (inconsistent + unit skipping)

In-game

In-game the sensitivity is a multiplier of windows + dpi. With rawinput (more information about rawinput down below) on the game will ignore the windows sensitivity, and act like it is set to 6/11, since it is 1:1. If you are unsure on using rawinput or not, then try and decide on what feels best.

Note that you can apply acceleration to your game through the in-game settings for keyboard & mouse, or by a third party program if this is what you want. It is advised to play without acceleration on, but everything is subjective. I suggest staying away from acceleration in-game since it is tied to the current FPS.

Choosing a mouse

Ambidextrous and ergonomic, there really are not cons or pros to ambidextrous and ergonomic, it is preference. Ambidextrous means you can use the mouse with your right or left hand with no discomfort .With ergonomic mice they are only designed for a certain hand (I.E Left or right hand.)

Grip

The way you’re holding the mouse is called mouse-grip, this is useful to know when you’re looking for a new mouse. We have 3 different types of grip:

·Palm ·Claw ·Fingertip

Palm grip, this grip used more commonly for ergonomic or larger mice. The user rests the entire palm on the mouse as well as the fingers

Claw grip is used for medium sized mice. The user holds the mouse with the fingertips and the base of the palm for support.

Fingertip grip is used for small mice. The user only holds the mouse with the fingertips which can lead to fatigue, be aware of that.

Myths

Acceleration

Generally in CS:GO, acceleration is considered bad, the argument is that it makes mouse movement inconsistent. Acceleration applies a filter to the mouse movement that changes its speed depending on how fast the mouse is moving. This makes it so you can change how much you’re turning with flicks. The acceleration filter in CS:GO is inconsistent, because the FPS is inconsistent. Even if you can achieve a stable FPS it is bound to drop at points therefore making it inconsistent. If you want to use acceleration I suggest using a third party application such as QL mouse filter. I personally believe you can be better with acceleration, but it takes longer and requires more effort. It is possible to have low sensitivity and still have the maneuverability from the higher ranges.

Rawinput – M_Rawinput

Rawinput is used to make the game take information from the mouse and bypassing the OS. Some people believe rawinput is bugged in CS:GO and this question still remains unanswered. Recent tests have showed it to be bugged, but still no confirmation from Valve about this issue. Test provided in annex.

In-depth explanation of Rawinput – Rawinput 0 resets the mouse cursor to the middle of the screen once every frame, higher FPS gives more resets, therefore better feel. Rawinput 1 makes CS:GO asks the Operating System which pointing device is being used and resets the cursor position as often as the polling rate allows it. If you aren't using Rawinput 1, I suggest capping your fps for more consistency. Rawinput is static no matter the fps.

Mark C

This mouse fix is for old games. Windows’ own acceleration curve (Enhanced Pointer Precision) would be applied in older games because the engine was trash, this fixes the curve so it always results in 0 acceleration. The old engines applied this acceleration by a mistake and newer engines and games do not have this mistake. Having this fix or not should make no difference.

What is aiming?

Flicking

Reactionary aim. Flicking is moving your mouse at a very fast speed to hit an enemy. You flick to the enemies that came as a surprise, perhaps flanked you or similar. This is mainly built on muscle memory and will be the difference from a great and a good player.

Tracing

Tracing or tracking is mainly used with pistols and spraying. If you are good at tracing, your crosshair will never leave the enemies head unless they go behind cover, in which case, pre aim kicks in and takes over. This is a mix of reaction and hand-eye coordination. It is determined by your own movement as well as the enemy’s which you’re tracking.

Preaim

This comes from deathmatch and experience, having your crosshair ready for the enemies to appear will eliminate travel time and thus reduce reaction time and it’s more consistent than always flicking to the enemies.

Movement

In most cases you have to adjust with your own movement to either counter the opponent's movement or to adjust your own mistake. It’s important to practice stutter stepping and peeking around corners properly.

Being in a position where you are exposed to 1 angle, or as few as possible, will help you focus on that one angle where the enemies could come. Peeking for information is also recommended, and then adjusting your aim afterwards to the new information, making you hit the opponent when peeking around the corner, without moving your mouse.

Training

Training should have focus points instead of being mindless, Deathmatch is pointless since all the angles are wrong, the situations is wrong. Its only purpose is to warm up, not to practice your aim.

Main focus point while wanting to improve aim: ·Tracking ·Flicking ·Muscle memory ·Preaim

When you’re practicing you need to concentrate on key points instead of just doing it, it’ll do nothing if you don’t focus on areas you want to improve upon.

These things are what makes your aim good. If you want to improve you have to push your own limits.

Aim_training_v2x is good for building muscle memory by flicking. Settings to use: Classic fast aim - size 12 delay 0.25 duration 0.50 targets 100 (These are the advanced settings and have been advertised by people like Friberg.)

Aim_botz is good for flicking and tracing.

Block off 3 out of 4 sections. Flick from 1 bot to the next and attempt to be as fast as possible while still maintaining accuracy. Don’t go faster than you can. Noclip out of the middle box and run around the bots holding your crosshair on their head while jumping and crouching.

Aimtraindriving is good for tracing. Follow the bots’ heads around for a second or two before killing them. Make sure to enable god mode with ‘god’ in console.

recoil_master if you want to practice recoil, I don’t... It is very self-explanatory.
